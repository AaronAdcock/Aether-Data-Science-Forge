# Contributing to Aether-Data-Science-Forge

Thank you for your interest in contributing to our enterprise MLOps platform!

## Code of Conduct
Please adhere to our standard professional code of conduct.

## Development Process
1. **Fork the Repository**: Create a personal fork of the project.
2. **Branching**: Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature`).
3. **Coding Standards**:
   - Follow PEP8 style guidelines.
   - Include Google-style docstrings for all functions and classes.
   - Use Pydantic for data validation where applicable.
4. **Testing**:
   - Add unit tests for all new functionality.
   - Ensure all integration tests pass via `pytest`.
5. **Pull Requests**:
   - Provide a clear description of the changes.
   - Reference any related issues.
   - Ensure CI/CD pipeline passes.

## Review Process
All submissions will be reviewed by the maintainers. We aim for a 24-48 hour turnaround on feedback.

## Questions?
Open an issue for any architectural discussions or bugs.
